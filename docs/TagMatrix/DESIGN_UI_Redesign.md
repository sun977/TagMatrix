# 设计文档 - 优化 TagMatrix 前端 UI 设计

## 架构概览

### 整体架构图
```mermaid
graph TD
    A[App.vue] --> B[Layout.vue 整体布局]
    B --> C[Sidebar 左侧导航栏]
    B --> D[Main Content 主内容区]
    B --> E[Global Settings 模态框 (含开发者模式开关)]
    B --> F[AI Copilot 常驻右侧边栏]
    B --> G[Status Bar 底部状态栏]
    D --> H[Dashboard.vue 概览]
    D --> I[DataSource.vue 数据源管理]
    D --> J[TagRuleConfig.vue 标签与规则配置]
    D --> K[TaskKanban.vue 打标任务看板]
    D --> L[TaggedData.vue 打标结果数据]
    D --> M[DatabaseAdmin.vue 系统数据库管理 (高阶)]
```

### 核心组件设计

#### Layout.vue
- **职责**：
  - 渲染左侧固定宽度的 Sidebar（深色背景 `#1e1e2d`），包含 Logo、核心路由菜单。
  - 渲染右侧自适应的主内容区，背景为浅灰 `#f5f6f8`，页面主体内容通常采用纯白 `#ffffff` 卡片包裹。
  - 渲染右侧可折叠的 **AI Copilot 全局副驾驶侧边栏**，生命周期跨越所有路由页面。
  - 渲染底部的 **Status Bar**（展示版本号、DB 状态、大模型 API 连通性）。
  - 提供全局快捷键唤起命令面板 (`Ctrl+K`) 以及唤起全局 Settings 弹窗的入口。

#### Dashboard.vue
- **职责**：数据大盘总览与任务执行监控。
- **结构**：
  - 顶部：欢迎语和副标题。
  - 数据区：4个带图标和点击跳转详情功能的数据统计卡片（总数据、打标覆盖率、标签数、规则数）。
  - 快捷操作区：前往“导入新数据”和“新建标签”的入口。
  - 任务监控区：基于 WebSocket 实时刷新（`taskProgress`）的近期打标任务表格，精准展示动态进度和真实耗时。

#### DataSource.vue
- **职责**：用户业务数据的导入、检索与清理。
- **结构**：
  - 顶部：数据集切换器与数据源过滤选项。
  - 工具栏：支持拖拽上传的导入区、全量或按过滤条件导出的按钮、清空当前数据集数据等。
  - 数据表：根据选中数据集的 Schema 动态生成的 `el-table`。

#### TagRuleConfig.vue
- **职责**：三栏式结构，管理数据集专属的标签与规则。
- **结构**：
  - **左栏 (数据集导航)**：列表展示所有系统数据集，控制中右栏数据的联动。
  - **中栏 (标签管理)**：在选中数据集下展示关联的标签列表（`el-tree` 或列表），支持增删改查。
  - **右栏 (规则配置器)**：展示当前选中标签的关联规则。
    - 规则可视化构建器：通过表单组合字段、操作符、阈值。
    - 抽样测试区 (Dry Run)：底部提供按钮，一键在弹窗中验证当前规则在真实数据中的命中情况。

#### TaskKanban.vue
- **职责**：打标任务的灵活发起与生命周期追踪。
- **结构**：
  - 顶部配置区：选择目标数据集、来源文件过滤、规则执行范围、执行策略（追加/覆盖）以及打标模式（多标签/单标签/混合）。
  - 历史任务区：带有工具栏（状态、时间过滤）的 `el-table`。
  - 列表呈现：秒级刷新的进度条、真实耗时。操作列提供查看命中日志（弹窗表格）、导出 CSV 和“一键回退 (Rollback)”功能。

#### DatabaseAdmin.vue [高级功能]
- **职责**：为高阶用户/实施人员提供的防崩底层 SQLite 运维入口（需在设置中开启“开发者模式”）。
- **结构**：
  - 顶部：醒目的高危操作警告栏。
  - **Tab 1：系统物理表模式**：严格只读 Schema，支持对 `sys_tags` 等物理表的内联编辑 CRUD，及带主键/时间戳清洗防护的安全导入/导出。
  - **Tab 2：用户业务数据模式**：以逻辑数据集为维度，将 `raw_data_records` 中的 `data` JSON 字段动态展平为多列虚拟表，提供双击修改业务字段并自动 JSON 序列化的安全编辑能力。
  - **Tab 3：SQL Console**：带语法高亮的 SQL 执行器，支持 SQLite 原生 JSON 提取查询，内置快捷 Snippets。
  - **Tab 4：备份还原**：一键生成全库 `.db` 快照，及带有二次危险确认的恢复机制。

#### Global Settings & AI Sidebar
- **SettingsDialog**: 左 Tab 右表单的模态框，包含通用（含主题切换）、AI 模型配置（Key/Base URL/连通测试）、Prompt 策略管理、以及包含路径查看和“开发者模式”隐藏开关的高级系统设置。
- **AI Copilot Sidebar**: 
  - 顶部包含极简模型切换器和“当前页面上下文感知”眼睛开关。
  - 中部是支持 Markdown 渲染（带 SQL 快捷复制/执行）和内联 Action Blocks 的对话流。
  - 具备未配置 API Key 时的优雅缺省引导界面。

## UI 规范定义
在 `main.scss` 中更新 CSS 变量以匹配现代化、清爽护眼的桌面端风格：
- `--tm-bg-sidebar: #1e1e2d` (深色侧边栏)
- `--tm-bg-main: #f5f6f8` (浅灰工作区背景)
- `--tm-bg-card: #ffffff` (内容卡片背景)
- `--tm-bg-hover: #f0f2f5` (悬浮高亮色)
- `--tm-accent-primary: #52c48f` (主色调薄荷绿)
- `--tm-border-radius: 8px` (组件圆角，卡片部分可能用 `12px`)
- 表格头部背景：`#f9fafc`
- (支持根据系统设置自动切换至 Dark Mode 变量集)
