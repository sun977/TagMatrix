# 系统数据库管理 (DatabaseAdmin) 开发计划

## 1. 概述
本项目旨在开发 TagMatrix 的“系统数据库管理 (DatabaseAdmin)”高级功能模块。该模块面向高阶用户或系统管理员，提供对底层 SQLite 数据库的全面控制能力，包含 SQL 查询终端、可视化表结构与数据管理、以及备份与还原中心。由于该功能涉及高危操作，默认处于隐藏状态，必须在全局设置中开启“开发者模式”后方可访问。

## 2. 阶段划分
整个开发周期划分为四个主要阶段，依次推进：
- **阶段一：基础骨架与入口控制** —— 搭建页面路由、开发者模式开关逻辑、页面基础布局及高危提示。
- **阶段二：SQL 查询终端** —— 实现带语法高亮的 SQL 编辑器、常用 SQL 模板库、结果集动态渲染。
- **阶段三：可视化表结构与数据管理** —— 实现系统物理表与用户业务数据的双模式切换及相应的虚拟/物理 CRUD 操作。
- **阶段四：备份与还原中心** —— 实现数据库快照管理、一键备份、高危恢复机制及数据库文件的导入导出。

## 3. 详细任务分解

### 阶段一：基础骨架与入口控制
- **前端任务**：
  - 在全局状态管理中（如 Pinia 对应的 store）添加 `developerMode` 状态及相应的开关控制逻辑。
  - 配置侧边栏导航，实现基于 `developerMode` 状态动态控制“系统数据库管理”入口的显示与隐藏。
  - 创建 `src/views/dataAdmin/DatabaseAdmin.vue` 作为主页面。
  - 在页面顶部添加明显的“危险操作/高级模式”高亮黄/红警示框。
  - 配置页面内三个主要功能模块的 Tab 切换布局（SQL终端、可视化管理、备份与还原）。
- **预期成果**：完成页面路由及基础布局搭建，实现根据设置动态显示页面入口。

### 阶段二：SQL 查询终端 (SQL Console)
- **前端任务**：
  - 引入并配置代码编辑器组件（如 Monaco Editor 或 CodeMirror），支持 SQL 语法高亮及基本的智能提示。
  - 在编辑器侧边或顶部实现“快捷代码片段 (Snippets)”组件，内置常用的 JSON SQL 提取/修改等高级查询模板，支持一键插入。
  - 构建“执行语句”按钮及结果展示区，利用动态数据表格（如 `el-table`）渲染 `SELECT` 的多态结果集。
  - 针对 `UPDATE`/`DELETE` 等操作，直观展示受影响行数及耗时；若语法错误，需高亮显示报错信息。
- **后端任务**：
  - 设计并实现 SQL 执行接口，接收原始 SQL 语句并在 SQLite 数据库中执行，返回规范化的结果集或详细的错误信息。

### 阶段三：可视化表结构与数据管理 (Table Explorer)
- **前端任务**：
  - 实现“系统物理表”与“用户业务数据”的双选项卡切换机制。
  - **Tab 1：系统物理表模式**：
    - 左侧栏展示底层物理表列表（如 `sys_tags`, `raw_data_records` 等，结构只读）。
    - 右侧栏渲染全量物理表数据，提供双击内联编辑、勾选删除、行尾新增等所见即所得的 CRUD 操作。
    - 提供物理表级别的导出（Excel/CSV）和导入功能。
  - **Tab 2：用户业务数据模式 (JSON 解析模式)**：
    - 左侧栏根据 `sys_datasets` 渲染用户创建的“业务数据集”列表（虚拟逻辑表）。
    - 选中数据集后，读取对应的 `schema_keys`，动态扁平化渲染 `raw_data_records` 中的 `data` 字段（JSON格式）为多列数据表。
    - 支持虚拟 CRUD，用户在单元格双击修改数据后，前端将其重新序列化打包为 JSON，并触发对应行的更新保存。
- **后端任务**：
  - 提供获取系统物理表及其 Schema 的接口。
  - 提供基于数据集动态获取、分页查询逻辑数据的接口。
  - 提供物理表级及业务级的 CRUD 数据保存接口（需处理 JSON 的安全合并与校验）。

### 阶段四：备份与还原中心 (Backup & Restore)
- **前端任务**：
  - 设计备份记录列表，展示历史快照（包括备份时间、文件大小、备注）。
  - 实现“一键创建备份快照”功能及前端反馈交互。
  - 针对快照恢复操作提供“从该备份恢复 (Restore)”按钮，点击后需强制触发二次高危安全验证弹窗。
  - 提供将当前 `.db` 文件导出至本地，以及从外部导入 SQLite 数据库文件的功能。
- **后端任务**：
  - 实现数据库备份快照的生成与列表获取接口。
  - 提供高危的恢复操作接口，用指定快照覆盖当前数据库，并实现服务的安全重载/重启以释放文件占用。
  - 提供物理 `.db` 文件的直接下载与上传接口。

## 4. 风险与注意事项
1. **权限与安全隔离**：该功能模块具备底层数据的直接控制权，极易导致数据损坏。务必确保入口严格被“开发者模式”限制。
2. **JSON 数据完整性**：在“用户业务数据模式”下修改虚拟列并序列化保存时，必须严谨处理，避免覆盖丢失未修改的其他 JSON 字段。
3. **高危恢复机制设计**：恢复或导入 `.db` 文件时，当前数据库可能处于被连接占用的状态。后端需要有完善的文件锁释放与重启机制，防止数据库文件被意外损坏。
4. **大结果集渲染性能**：SQL 终端执行 `SELECT` 时，若查询结果过多可能导致前端页面卡死。建议前端默认开启结果集条数限制或分页，并给予明显提示。
