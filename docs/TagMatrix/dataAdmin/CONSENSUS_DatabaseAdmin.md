# 共识文档 - DatabaseAdmin系统数据库管理

## 需求说明与边界
本项目开发“系统数据库管理”功能，主要面向高级用户/开发者。包含三个子模块：
1. **SQL 查询终端**：允许用户编写 SQL 查询和操作底层 SQLite 数据库，并提供结果集展示。
2. **可视化表结构与数据管理**：提供系统物理表的查看和增删改查；提供基于 `raw_data_records` 中 `data` 字段 JSON 解析的虚拟表管理，支持内联编辑业务数据。
3. **备份与还原中心**：提供快照式数据库备份、导出和导入恢复机制。

**明确边界**：
- 仅通过“开发者模式”限制入口，无复杂角色权限。
- 仅管理当前应用内置 SQLite 数据库，不连接外部数据库。

## 技术实现方案与共识约束
1. **代码编辑器方案**：
   - 采用轻量级的 **CodeMirror**（通过 `vue-codemirror` 或原生封装）作为前端 SQL 编辑器组件，以提供基础的 SQL 语法高亮和输入体验。
2. **数据库高危恢复锁定与重载机制**：
   - Go 后端将专门提供一个接口 `RestoreDatabase(backupPath string)`，其内部逻辑严格遵循：**断开现有数据库连接 -> 替换/覆盖 .db 文件 -> 重新建立连接**。
   - 前端在触发此操作前必须给予用户明确的高危警告弹窗（例如：“此操作将覆盖当前所有数据，且应用将自动重启，是否继续？”）。
   - 恢复接口返回成功后，前端主动调用 Wails 提供的窗口重载方法（如 `WindowReload`）或执行路由刷新，重新加载整个应用状态。

## 验收标准
1. **安全性**：危险操作（如 DELETE/UPDATE、恢复备份）均有显著警示及二次确认。
2. **功能性**：
   - CodeMirror 正常渲染且支持 SQL 高亮。
   - JSON 数据的解析展示与内联修改能够准确回写且不损坏未修改字段。
   - 数据库释放并覆盖的流程稳定，无文件锁定报错。
