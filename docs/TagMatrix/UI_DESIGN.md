# 前端 UI 设计与规划文档 (TagMatrix UI Design)

## 1. 整体设计风格 (Design System)

*   **视觉风格**: 极简、清爽的现代化数据管理后台布局。
    *   **主色调 (Primary Accent)**: 柔和的薄荷绿/翠绿色 (类似 `#52c48f`)，用于主按钮、导航高亮和强调图标，营造高效、护眼的工作体验。
    *   **背景色**: 左侧导航栏采用深色主题 (`#1e1e2d`)，主内容区采用极浅的灰色 (`#f5f6f8`) 和纯白色 (`#ffffff`) 的卡片。
    *   **边角设计**: 大量使用圆角 (`border-radius: 8px` 至 `12px`)，包括卡片、按钮和对话框，增加界面的亲和力。
    *   **留白 (Whitespace)**: 增加内容与边界的呼吸感，通过轻微的背景色差和柔和的阴影区分层级，而非生硬的边框线。
*   **组件库**: `Element Plus` (Vue 3)。通过 CSS 变量和自定义样式覆盖默认主题，深度定制薄荷绿主色调和组件外观。
*   **图标库**: `@element-plus/icons-vue`。
*   **布局骨架**:
    *   **左侧导航栏 (Sidebar)**: 固定宽度，深色背景。包含 Logo、系统名称、核心功能路由导航项，底部包含全局“设置”入口。
    *   **中央工作区 (Main Content)**: 占据主要视口，浅灰背景，渲染对应的页面视图（路由）。每个页面拥有自己独立的头部标题和操作区。

## 2. 核心页面规划 (Views)

### 2.1. 概览控制台 (Dashboard)
*   **数据统计卡片**: 展示当前库内总数据量、已打标数据量及覆盖率、系统标签总数、规则总数。点击卡片可唤起对应弹窗查看各数据集的明细统计。
*   **快速操作**: 提供前往“导入新数据”和“新建标签”的快捷卡片入口。
*   **最近打标任务**: 简略展示最近 5 次的打标批次及状态。集成了 WebSocket 实时通信，任务执行状态、进度和耗时会秒级动态跳动更新。

### 2.2. 数据源管理 (DataSource)
*   **数据集管理**: 左侧或顶部下拉框支持数据集切换，所有数据均归属于特定数据集。
*   **导入区**: 支持通过本地文件选择或拖拽方式，上传 Excel/CSV 文件导入数据，并可配置数据来源标识（Source File）。
*   **数据预览表 (Data Table)**: 动态表格展示，根据当前数据集的 Schema 动态生成列头。支持基于字段值、来源文件进行组合条件检索和分页浏览。
*   **清理功能**: 提供清空当前数据集或清空特定来源文件数据的功能。

### 2.3. 标签与规则配置 (TagRuleConfig)
这是一个三栏式 (Three-pane) 的高度交互复杂页面：
*   **左侧栏 (数据集选择)**: 列表展示当前所有的系统数据集，选中某个数据集后，中右侧面板的数据会随之联动。
*   **中间栏 (标签列表与管理)**: 展示当前数据集下的所有标签。支持新增标签（分配颜色）、编辑、删除。点击特定标签后，右侧展示该标签关联的规则。
*   **右侧栏 (规则配置器)**: 
    *   展示和管理选中标签下的所有匹配规则列表。
    *   **规则可视化构建器 (Rule Builder)**: 允许用户以表单形式组合多条判定条件（字段 + 关系符 + 阈值），支持 AI 辅助生成规则（自然语言转配置）。
    *   **快速测试 (Dry Run)**: 规则配置底部提供“抽样测试”按钮，在弹窗中展示抽样的真实数据，直观验证当前规则的命中情况和准确性。

### 2.4. 打标任务看板 (TaskKanban)
*   **发起新任务区域**: 
    *   灵活配置任务参数：选择目标数据集、选择来源文件过滤（或全量）、选择要执行的规则范围（全部或特定规则）。
    *   **执行策略**: 支持“追加模式”（保留原有标签）和“覆盖模式”（清除原有标签）。
    *   **打标模式**: 支持“多标签模式”（命中均打标）、“单标签模式”（仅取最高优先级）和“混合模式”（区分主副标签）。
*   **任务历史表**: 
    *   展示所有历史任务的详细记录：批次名、状态、进度条、处理条数、精准耗时、创建时间。
    *   **实时刷新**: 运行中的任务通过 WebSocket 接收进度，页面动态更新毫无延迟。
    *   **操作列**: 支持查看详细命中日志（弹窗表格）、导出日志为 CSV、删除任务记录，以及安全可靠的“一键回退 (Rollback)”功能以撤销错误打标。

### 2.5. 打标结果数据 (TaggedData)
*   **多维检索与过滤**: 顶栏提供强大的查询能力，可选择数据集，根据关键字、标签分类、特定打标批次进行组合过滤。
*   **彩色高亮数据表**: 动态表格中专门独立一列展示该条数据所带有的标签（呈现为对应的彩色 Tag 组件），打标结果一目了然。
*   **详情与导出**: 支持查看单条数据的详细打标来源（人工/规则/AI）；提供导出当前过滤结果为 Excel/CSV 的功能。

### 2.6. 全局设置 (SettingsDialog)
*   **弹窗形态**: 位于左下角的设置入口唤起。为了容纳不断增加的配置项，弹窗内部采用左侧 Tab 栏导航、右侧表单配置的经典面板布局。
*   **通用设置 (General)**: 
    *   **主题与外观**: 切换系统亮色 (Light)、暗色 (Dark) 模式，或跟随操作系统状态。
    *   **消息通知**: 开启/关闭后台任务（如打标批次完成、导出完成）的应用内 Toast 提醒或系统级推送。
*   **AI 模型配置 (AI Provider)**:
    *   **服务商与网络**: 用于配置各类大模型（如 OpenAI、千问、DeepSeek 或 Ollama 本地服务）的 API Key 和 Base URL。
    *   **模型指定与连通性**: 指定调用的模型版本（Model Name），并提供一个“测试连接 (Ping)”按钮以验证网络与 Key 的有效性。
*   **Prompt 与策略 (Prompts)**:
    *   **系统提示词管理**: 允许高级用户微调系统预设的 System Prompt，用于优化 AI 在“自然语言生成规则”、“AI 意图打标”等不同场景下的表现。
*   **高级与系统 (Advanced & System)**:
    *   **本地存储信息**: 只读展示当前 SQLite 数据库 (`data.db`) 及应用运行日志所在的本地绝对路径，并提供“在文件资源管理器中打开”的快捷按钮。
    *   **开发者模式开关 (Developer Mode)**: 在该面板底部，提供一个需特定交互（例如连续点击系统版本号 5 次）才能触发的隐藏开关。开启后，左侧主导航栏才会露出【系统数据库管理】等高危模块入口。

### 2.7. 系统数据库管理 (DatabaseAdmin) [高级功能]
*入口隐藏逻辑：此功能默认隐藏，必须在全局设置中开启“开发者模式 (Developer Mode)”后，才能在左侧导航栏底部看到入口。*
*此功能面向高阶用户或系统管理员，提供对底层 SQLite 数据库的全面控制能力。页面顶部需带有明显的“危险操作/高级模式”警告提示（高亮黄/红警示框）。*
*   **SQL 查询终端 (SQL Console)**: 
    *   **输入区**: 提供一个支持 SQL 语法高亮的编辑器（可集成简单的关键词提示）。
    *   **JSON 查询支持 (User Data SQL)**: 明确提示高阶用户，虽然业务数据打包在 `raw_data_records` 的 `data` 字段中，但底层的 SQLite 完全支持原生的 JSON 提取函数。用户完全可以通过如 `SELECT json_extract(data, '$.文章标题') FROM raw_data_records WHERE dataset_id = 1` 或更简写的 `data->>'文章标题'` 语法来直接使用 SQL 查询和过滤业务数据。
    *   **快捷代码片段 (Snippets)**: 为了降低用户手写 JSON SQL 的门槛，在编辑器侧边或顶部提供“常用 SQL 模板”库（例如：如何按 JSON 字段条件筛选、如何更新 JSON 内的某个 key 等），一键插入到编辑器中。
    *   **执行与结果**: 提供“执行语句”按钮，下方区域使用动态数据表格 (`el-table`) 渲染 `SELECT` 的结果集，或展示 `UPDATE`/`DELETE` 等操作的受影响行数及耗时。如果语法错误，醒目展示报错信息。
*   **可视化表结构与数据管理 (Table Explorer)**:
    *   **双模式切换 (Tab 切换)**: 顶部提供“系统物理表 (System Tables)”和“用户业务数据 (User Datasets)”两个选项卡。
    *   **Tab 1：系统物理表模式**:
        *   **左侧栏**: 展示底层真实的 SQLite 物理表（如 `sys_tags`, `raw_data_records`, `tag_task_batches` 等）。注意：表结构 (Schema) 严格只读。
        *   **右侧栏**: 呈现全量物理表数据。提供所见即所得的 CRUD（双击内联编辑、勾选删除、行尾新增）。
        *   **系统表导入导出**: 支持物理表级别导出为 Excel/CSV 和导入更新，内置 `id` 和时间戳的清洗防护，屏蔽脏数据写入。
    *   **Tab 2：用户业务数据模式 (JSON 解析模式)**:
        *   **背景逻辑**: 用户的真实业务数据实际上打包存在物理表 `raw_data_records` 的 `data` 字段（JSON 格式）中，直接通过物理表操作 JSON 非常反人类。
        *   **左侧栏 (逻辑表导航)**: 这里不展示物理表，而是根据 `sys_datasets` 渲染出用户创建的“业务数据集”列表（这相当于虚拟的逻辑表）。
        *   **右侧栏 (动态数据渲染)**: 选中某个数据集后，系统会读取其对应的 `schema_keys`，将 `raw_data_records` 中的 `data` JSON 字段扁平化展开为多列数据表。
        *   **虚拟 CRUD 操作**: 同样支持双击单元格内联编辑。当用户修改虚拟列的值时，前端在保存时将当前行的各列重新序列化打包成 JSON，并对 `raw_data_records` 触发行级别的 `UPDATE`。支持新增虚拟记录和物理删除记录。
*   **备份与还原中心 (Backup & Restore)**:
    *   **防崩保险**: 考虑到高级操作极易把数据库“玩崩”，提供独立的安全面板管理 `.db` 文件。
    *   **创建与列表**: 提供“一键创建备份快照”按钮。列表展示历史备份快照记录（备份时间、文件大小、备注）。
    *   **恢复机制**: 针对每条备份记录，提供“从该备份恢复 (Restore)”按钮（点击后需二次高危弹窗确认并重启服务生效），同时支持将当前的 `.db` 导出至本地或从外部导入 SQLite 数据库文件。

### 2.8. 全局智能助手 (AI Copilot Sidebar) [常驻侧边栏]
*入口逻辑：作为系统的“全局副驾驶”，该模块常驻在系统主窗口的最右侧。支持通过快捷键或折叠按钮自由展开/隐藏，其生命周期与主路由页面无关（从 A 页面切到 B 页面，AI 聊天记录与面板状态保持不变）。*
*   **面板头部 (上下文与执行策略)**:
    *   **模型切换器 (Model Switcher)**: 提供极简下拉菜单，支持在全局设置好 API Key 的前提下，快捷切换当前对话使用的具体模型版本（如在 `GPT-4o` 与 `Claude-3.5-Sonnet` 之间快速切换，适应不同复杂度的任务）。
    *   **当前上下文感知开关 (Context Awareness)**: 提供一个“眼睛”或“聚焦”图标开关。开启时，AI 将自动读取用户当前停留的左侧页面数据（如当前选中的标签、当前展开的数据集表格）作为背景提示词；关闭时，AI 仅保持全局的对话记忆。
    *   **系统指令快捷注入 (Prompt Inject)**: 提供快捷按钮，允许用户在当前对话会话中，临时注入一段强制性的 System Prompt（例如：“接下来请只输出 SQLite 的 JSON 提取 SQL 语句，无需任何解释”）。
*   **主对话流区 (Chat Stream)**:
    *   **多模态气泡流**: 类似 ChatGPT 的对话气泡。支持 Markdown 全量渲染，特别是对 SQL 语句、JSON 配置块提供高亮以及“一键复制”和“一键执行”按钮。
    *   **动作组件 (Action Blocks)**: AI 不仅能输出文本，当识别到用户的特定意图（如“帮我备份数据库”、“生成新规则”）时，能在气泡中直接渲染出薄荷绿的交互按钮。用户点击按钮，AI 面板便直接调用 Wails 底层命令完成操作，实现“对话即交互”。
    *   **配置占位与引导**: 若用户首次打开并未在【全局设置】中配置大模型 API Key，对话流区将呈现优雅的空状态占位图，并提供一个醒目的 `立即配置 AI 引擎` 按钮，点击直接呼出对应的全局设置弹窗 Tab。
*   **底部输入区 (Input Box)**:
    *   支持多行输入、回车发送/Shift+回车换行。
    *   支持简单的 `/` 快捷指令唤出内置的高频提问模板。
*   **多智能体支持 (Agent Hub / Custom Agents) [未来扩展]**:
    *   **概念规划**: 随着系统深入，单一的“打标副驾驶”可能不够用。建议未来在 AI 面板顶部设计一个“Agent 切换器”。用户可以定义多个专精于不同任务的 Agent（如：`SQL 专家`、`正则表达式大师`、`电商数据清洗助手`）。
    *   **Agent 定义要素**: 用户可以在【全局设置 -> Prompt 与策略】中新建 Agent，定义其头像、System Prompt、默认使用的大模型，甚至是能够调用的底层工具权限（Tools/Functions）。
    *   **交互形态**: 在侧边栏中切换 Agent，就相当于切换了一个全新的工作上下文和预设提示词环境，让 AI 更加收敛和专业。

## 3. 全局交互与用户体验 (Global UX)

*   **状态栏 (Status Bar)**: 在系统窗口最底部固定一条极简的细长状态栏。左侧显示当前 App 版本号和 SQLite 数据库的本地存储路径/状态；右侧显示当前选中 AI 大模型的连通性（如：绿色圆点表示 API 正常，红色表示 Key 异常或网络断开）。
*   **全局消息与后台通知 (Notification)**: 因为打标任务可能耗时较长，用户可以离开看板去配置其他规则。当后台打标任务完成或执行失败时，系统右上角（或调用 Wails 原生系统通知）弹出 Toast 消息，点击可直接跳转至【打标任务看板】。
*   **全局命令面板与快捷键 (Command Palette)**: 为高阶用户提供类似 VS Code 的 `Ctrl+K` (或 `Cmd+K`) 唤起全局搜索弹窗，快速定位并跳转到某个特定的标签配置页、数据集或系统设置面板；常用操作支持快捷键（如 `Ctrl+S` 保存规则/SQL）。
*   **亮暗色主题自适应 (Dark/Light Mode)**: 作为一个现代化的桌面效率工具，系统内置亮色与暗色两套主题。支持在【全局设置】中手动切换，或直接选择“跟随系统 (Auto)”，满足开发者夜间高强度操作的护眼需求。

## 4. 前端工程化与架构

*   **框架**: `Vue 3` (Composition API, `<script setup>`) + `Vite` 构建工具。
*   **路由**: `vue-router`，使用 Hash 模式保证与 Wails 桌面端容器的完美兼容。
*   **状态与通信**: 主要利用 Vue 响应式 `ref`/`reactive` 结合组件传参。大量使用 Wails 生成的 JS Bindings 直接调用 Go 后端方法。
*   **实时更新**: 使用 Wails 的 `EventsOn` 机制监听后端抛出的 WebSocket 事件（如 `taskProgress`），实现打标进度与时间的毫秒级响应。
*   **样式预处理器**: `SCSS`，允许嵌套语法、变量复用，维护各组件局部和全局统一的样式。
