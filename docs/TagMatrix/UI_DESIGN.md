# 前端 UI 设计与规划文档 (TagMatrix UI Design)

## 1. 整体设计风格 (Design System)

*   **视觉风格**: 极简、类 Notion / ChatGPT 的清爽布局。
    *   **主色调 (Primary Accent)**: 柔和的薄荷绿/翠绿色 (类似 `#52c48f` 或 `#34a853`)，用于主按钮和强调图标。
    *   **背景色**: 左侧导航栏采用极浅的灰色 (`#f7f7f9` 或 `#fcfcfc`)，主内容区采用纯白色 (`#ffffff`)。
    *   **边角设计**: 大量使用圆角 (`border-radius: 12px` 左右)，包括卡片、按钮和导航项，营造亲和力。
    *   **留白 (Whitespace)**: 增加内容与边界的呼吸感，去除多余的边框线，用轻微的背景色差或极淡的阴影区分层级。
*   **组件库**: `Element Plus` (Vue 3)。通过 CSS 变量覆盖其默认的蓝色主题，改为我们的薄荷绿主题。
*   **图标库**: `@element-plus/icons-vue`。
*   **布局骨架**:
    *   **左侧导航栏 (Sidebar)**: 宽度可调整，浅灰背景。导航项为圆角矩形，悬浮和选中状态呈现稍深的灰色 (`#e5e5e5`)。
    *   **中央工作区 (Main Content)**: 纯白背景，渲染对应的路由视图。内容居中或有合理的左右边距。
    *   **右侧面板 (Right Panel/Drawer)**: 类似参考图中右侧的“文件”或 AI 对话流，可以折叠隐藏，背景同样为极浅灰色，与主内容区分开。
    *   **顶部顶栏 (Header)**: 显示当前模块名称、全局任务状态跑马灯、以及呼出“全局设置 (配置 API Key 等)”的按钮。

## 2. 核心页面规划 (Views)

### 2.1. 首页 / 概览控制台 (Dashboard)
*   **数据统计卡片**: 当前总数据量、已打标数据量、标签总数、规则总数。
*   **快速入口**: 快速导入数据、快速新建标签。
*   **最近任务**: 简略展示最近 5 次的打标批次及状态。

### 2.2. 数据源管理 (Data Import/Export)
*   **上传区**: 支持拖拽的超大虚线框，用于上传 Excel/CSV。
*   **数据预览表 (Data Table)**: 使用 `el-table`。因为数据列是动态的，需要根据导入的数据动态生成列头。提供分页组件。
*   **导出功能**: 顶部工具栏提供“导出当前过滤条件下的结果”按钮。

### 2.3. 标签与规则配置 (Tag & Rule Config)
这是一个左右分栏 (Split-pane) 的复合页面：
*   **左侧 (标签树)**:
    *   使用 `el-tree` 展示标签的层级结构。
    *   支持右键菜单（新增子标签、删除、修改颜色）。
*   **右侧 (规则编辑器)**:
    *   当点击左侧某个标签时，右侧展示关联的匹配规则列表。
    *   **规则可视化构建器 (Rule Builder)**: 一个复杂的交互区域，允许用户使用下拉框组合 `字段 + 操作符 + 值`，支持 AND/OR 嵌套。
    *   **试运行面板 (Dry Run)**: 规则配置底部有一个“测试此规则”按钮，点击后在下方表格展示抽样的 10 条真实数据，直观展示命中情况。

### 2.4. 打标任务看板 (Task Kanban)
*   **发起任务**: 选择要执行的规则，输入批次名称，选择打标模式（主副标签），点击“开始执行”。
*   **任务历史表**: 展示历史 `TagTaskBatch`。包括状态（Loading 动画）、耗时、处理条数。
*   **操作列**: 提供“查看详细日志 (Audit Log)”按钮和危险操作的“一键回退 (Rollback)”按钮。

### 2.5. 全局设置面板 (Settings Dialog)
*   一个模态框，用于设置 `AppConfig`。
*   输入 OpenAI API Key、BaseURL、选择模型。
*   **Prompt 编辑器**: 允许用户自定义 AI 的 System Prompt。

### 2.6. AI 智能助手 (AI Assistant Sidebar)
*   **呼出方式**: 右下角的悬浮按钮（如带有闪烁特效的机器图标）。
*   **界面形态**: 类似 ChatGPT 的对话气泡流。
*   **功能**:
    *   底部有一个输入框，发送 `ChatWithAI` 请求。
    *   AI 返回的内容支持 Markdown 渲染（特别是 SQL 语句的高亮和一键复制按钮）。

### 2.7. 打标数据看板 (Tagged Data)
*   **顶部筛选栏 (Filter Bar)**: 提供多维度的组合查询（关键字搜索、标签分类筛选、批次筛选、打标状态等）。
*   **结果数据表 (Data Table)**: 使用 `el-table` 展示带有彩色高亮标签列的数据，方便直观查看打标效果。
*   **导出功能**: 顶部包含“导出数据”按钮，用于导出当前筛选条件下的已打标数据。

## 3. 前端工程化

*   **路由**: `vue-router`，使用 Hash 模式（Wails 推荐）。
*   **状态管理**: `Pinia`。用于全局存储当前的标签树状态、任务队列状态，避免组件间深层传递。
*   **样式预处理器**: `SCSS`，方便统一定义主题变量。

## 4. 下一步开发路径

1.  初始化 Element Plus, Vue Router, Pinia。
2.  搭建 `App.vue` 中的基础 Layout（侧边栏、顶栏、主内容区）。
3.  生成 Wails 的 JS Bindings，封装通用的 API 调用服务。
4.  逐个开发上述视图组件。
