# PR & 需求对齐文档 - TagMatrix 数据打标系统

## 1. 原始需求
这是一个给数据打标签的系统，用户可以自定义标签和标签属性，可以定义标签的打标规则，然后系统根据标签和打标规则匹配数据，将数据打上对应的标签。系统有UI界面，可以图形化操作。预留AI能力接口，让系统可以调用AI模型进行分析辅助用户打标（或者代替打标），AI的协议至少需要支持openai协议。

**具体功能点：**
- 自定义部分：标签名称、标签属性（说明）、打标签的规则（支持多字段匹配）、数据源支持（Excel, CSV, SQLite，预留 MySQL/PostgreSQL 扩展）。
- 打标模式：支持单标签模式、多标签模式、混合模式（主副标签）。
- 自动化部分：自动匹配数据、自动打标、自动记录详细日志（包含命中规则和打标理由）。
- 任务与版本控制：支持打标任务版本记录和一键回退。
- 规则调优：支持算子/规则的“试运行（Dry Run）”功能。
- 输入输出：数据导入导出，以及标签结构、匹配规则的 JSON/CSV 导入导出。
- AI 功能：除了辅助打标，提供 AI 对话窗口辅助数据筛选、生成 SQL 及工具使用指导。

## 2. 项目上下文与技术栈建议

基于您的反馈（单体可执行文件、几百万行数据、轻量级），我们调整了技术栈选型方案，以最适合构建本地桌面应用：

### 技术栈选型方案
- **应用框架**：`Wails` (Go + Vue3)。Wails 允许将 Go 后端和 Vue3 前端打包成一个单一的跨平台可执行文件（如 `.exe`），完全满足“本地运行、有UI界面”的需求，无需用户额外部署。
- **前端框架**：`Vue 3` + `Element Plus`。提供现代化的数据管理后台和规则配置 UI。
- **数据库**：`SQLite` + `GORM`。作为单文件数据库，完美契合本地应用的需求。SQLite 完全有能力支撑几百万行的数据体量（配合合理的批量插入和索引）。
- **任务处理**：使用 Go 原生的 `Goroutine` + `Channel` 构建本地内存级异步任务队列，无需引入 Redis 等沉重中间件。
- **AI 接入**：使用 `sashabaranov/go-openai` 库对接支持 OpenAI 协议的 LLM。
- **规则引擎**：复用原有 `NeoScan` 项目中的 `matcher` 引擎，支持多层级嵌套的布尔逻辑匹配。

### 现有资产复用
- **`matcher` 包**：现有的高性能、无状态的通用规则匹配引擎，支持 19 种操作符和嵌套逻辑，完美契合“定义打标签的规则”。
- **`tag_system_model`**：借鉴原有的 `SysTag`（标签树状结构）、`SysMatchRule`（标签规则）和 `SysEntityTag`（实体标签关联），并将其适配为 SQLite 模型。

## 3. 需求理解与功能边界

### 核心功能模块
1. **数据源与配置管理 (Data & Config Management)**
   - 支持上传 Excel/CSV 文件，统一集中入库至本地 SQLite。预留底层对 MySQL/PostgreSQL 的支持接口。
   - 支持标签体系、匹配打标规则的 CSV/JSON 格式导入导出，方便配置在多环境间迁移。
2. **标签与规则管理 (Tag & Rule Management)**
   - 标签 CRUD：支持层级结构，支持定义单标签、多标签、混合模式（主/副标签）。
   - 规则 CRUD：基于 `matcher` 构建规则树，支持多字段交叉匹配逻辑。
   - **试运行 (Dry Run)**：配置算子或规则时，支持针对部分数据进行结果预览调优，防止误操作导致错误打标。
3. **打标任务与版本引擎 (Tagging & Versioning Engine)**
   - 手动触发打标任务，使用 Goroutine worker pool 高效流式读取与评估数据。
   - **任务版本与回退**：每次打标生成唯一任务批次记录，若对打标结果不满意，支持基于该记录进行一键数据回退（Rollback）。
4. **AI 智能助手 (AI Assistant)**
   - **独立打标流**：调用大模型进行意图识别或长尾数据兜底打标。
   - **对话与分析交互**：提供 AI 聊天窗口，帮助用户分析数据特征、生成查询 SQL 语句，或直接充当软件操作指南解答问题。
5. **结果输出与精细化日志 (Output & Audit)**
   - 在 UI 查看打标结果并导出“数据+关联标签”的 Excel/CSV。
   - **打标追踪溯源**：记录极详尽的日志（包含数据命中的具体规则、匹配算子、计算方式和最终打标理由），让用户能清晰判断打标的合理性。

### 明确不包含（Out of Scope）
- [ ] 复杂的 ETL 数据清洗（系统仅关注打标，假设输入数据已初步清洗）。
- [ ] 外部大型数据库（如 MySQL/Oracle）直连（目前仅支持文件导入到本地 SQLite）。

## 4. 最终共识与验收标准

经过沟通，我们达成以下架构共识：
- **产品形态**：基于 Wails 构建的跨平台单体桌面应用程序（最终交付 `.exe` 等二进制文件）。
- **数据管理**：文件导入后统一入库 SQLite，在本地构建轻量级数据中心，最高支持数百万级别数据。
- **AI 整合**：AI 将作为一个独立的任务流，配合规则引擎实现“规则首发，AI 兜底”的混合打标模式。

### 功能验收
- [ ] 能够作为单体桌面应用启动，展示 Vue3 UI 界面。
- [ ] 能成功上传包含至少百万行的 CSV/Excel 文件，存入本地 SQLite。
- [ ] 能在 UI 上创建规则，并支持配置阶段的“试运行（预览结果）”。
- [ ] 支持单标签、多标签、混合打标模式切换。
- [ ] 支持导入导出标签体系和规则配置（JSON/CSV）。
- [ ] 打标任务生成清晰的版本批次，并支持一键回退到打标前状态。
- [ ] 能查看包含“打标理由和触发规则”的详尽日志。
- [ ] 具备 AI 聊天窗口，能够辅助用户解答工具使用方法、分析数据、生成针对当前 SQLite 表结构的查询 SQL。

### 质量验收
- [ ] 百万级数据导入和打标过程中，UI 不卡顿（异步执行）。
- [ ] 架构复用：成功接入并运行原 `NeoScan` 的 `matcher` 引擎。
- [ ] 代码整洁：Go 和 Vue 侧的交互接口定义清晰。
