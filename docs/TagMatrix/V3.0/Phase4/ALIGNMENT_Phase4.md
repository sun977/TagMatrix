# 需求对齐文档 - 局部资产导出 (dataset_with_rules.json)

## 原始需求
Phase 4: 导入导出资产解耦 (数据中台化)
*   **局部资产导出**：在数据集管理中，提供“数据集结构 + 专属打标规则”的打包导出功能 (`dataset_with_rules.json`)，实现业务资产与工程代码的物理分离流转。

## 项目上下文

### 技术栈
- 编程语言：Go
- 框架版本：Wails, Vue3 + Element Plus
- 数据库：SQLite (GORM)

### 现有架构理解
- **全局标签 (SysTag)**：通过 `Path` (如 `/A/B/`) 标识层级，已经实现了 `tags.json` 的导出导入。
- **数据集 (SysDataset)**：包含数据集名称、描述、Schema (表头字段)。
- **规则 (SysMatchRule)**：绑定在特定 `DatasetID` 和 `TagID` 上的打标逻辑 (JSON格式)。
- 局部资产导出旨在将特定数据集及其所有的专属规则作为一个业务资产整体导出，方便在其他环境中直接导入，复用业务知识。

## 需求理解

### 功能边界
**包含功能：**
- [x] 后端 `dataset_service` 或 `dataimport_service` 提供导出和导入 `dataset_with_rules.json` 的 API。
- [x] 导出时：将数据集基础信息，以及该数据集下所有的规则打包，其中规则与标签的关联关系通过标签的 `Path` (层级名称路径) 来解耦，从而不依赖具体的 `TagID` 数据库自增主键。
- [x] 导入时：解析 JSON，如果数据集不存在则创建；遍历规则，根据 `tag_path` 在当前数据库中寻找标签，如果标签存在则将规则挂载（不存在则报错或跳过）。
- [x] 前端：在【数据集管理】界面为每个数据集提供“导出业务资产”按钮；并在顶部或合适位置提供“导入业务资产”按钮。

**明确不包含（Out of Scope）：**
- [x] 导入时自动创建全局标签。因为 Phase 4 的“全局资产导出”已经实现了标签树的流转，局部资产导入假设目标环境已经拥有了对应的全局标签（或手动创建了对应的标签层级）。

## 疑问澄清

### P0级问题（必须澄清）
1. 问题描述：导入时如果目标标签 (`TagPath`) 在当前环境中不存在，如何处理？
   - 建议方案：跳过该规则的导入，并可以在导入完成后向用户提示“X条规则因标签不存在被跳过”。这符合“全局资产(tags)与局部资产(dataset_with_rules)解耦”的设计理念。

## 验收标准

### 功能验收
- [x] 能将指定的数据集及其所有关联的规则导出为一个干净的 JSON 文件。
- [x] 导出的 JSON 不包含数据库环境特定的主键 ID (除了必要的业务名称、Path等)。
- [x] 能将导出的 JSON 重新导入到系统中，如果同名数据集存在，可选择合并或更新；规则能正确挂载到对应标签上。
- [x] 前端有对应的导出和导入入口，操作有成功提示。
