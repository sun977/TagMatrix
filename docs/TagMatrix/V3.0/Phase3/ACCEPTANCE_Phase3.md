# 验收记录 - Phase 3: 打标任务引擎与看板改造

## 功能验收清单

### 1. 后端打标引擎隔离 (前期已顺带完成)
- [x] **数据集级规则隔离**：`TaskEngineService.RunTaggingTask` 执行时，通过 `dataset_id` 从 `sys_match_rules` 表拉取专属规则。
- [x] **数据集级数据运算**：在批量拉取原始数据 (RawDataRecord) 时，SQL 查询严格附加了 `WHERE dataset_id = ?` 条件，避免数据错乱。

### 2. 前端打标任务看板 (本期核心增强)
- [x] **发起打标界面增强**：
  - “选择目标数据集”已作为必填项实现。
  - 选择数据集后，级联动态拉取对应的规则 (`availableRules`) 和来源文件选项。
- [x] **任务历史展示优化**：
  - 任务列表表格新增 **目标数据集** 列。
  - 后端返回的底层 `dataset_id` 已在前端通过预先拉取的 `availableDatasets` 完成内存映射，显示为中文数据集名称。
- [x] **任务过滤交互**：
  - 顶部筛选器新增 **按数据集过滤** 的下拉框。
  - `filteredTaskHistory` 计算属性已适配新增的数据集过滤逻辑，能够与时间、状态过滤器实现交叉复合查询。
  - 切换数据集过滤器时，自动将分页重置为第一页。

## 验收结论
**Phase 3 重构目标全部达成。**
引擎层与展示层均已闭环支持了基于 `dataset_id` 的任务边界隔离，确保了全局任务不跨界、历史记录可追溯，为后续的数据集业务化解耦（Phase 4）打下了坚实基础。
