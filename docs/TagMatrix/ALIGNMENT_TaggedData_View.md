# 需求对齐文档 - 新增打标数据看板页面 (TaggedData_View)

## 原始需求
用户指出 `UI_DESIGN.md` 中的核心页面规划部分缺少了一个用于查看“打完标签后的数据”的页面。作为一个数据打标系统，确实需要一个独立的页面来检索、查看、过滤和导出已完成打标的数据。

## 项目上下文

### 技术栈
- 编程语言：Vue 3 + TypeScript
- 框架版本：Element Plus
- 路由：Vue Router (Hash 模式)
- 样式：SCSS (遵循薄荷绿主色调、极简圆角卡片风格)

### 现有架构理解
- 架构模式：采用 `Layout.vue` 作为主框架，左侧为可折叠可拖拽宽度的侧边栏，右侧为主内容区。
- 核心模块：目前已有 `Dashboard`、`DataSource`、`TagRuleConfig`、`TaskKanban`。
- 集成点：新页面需要作为新的路由子节点注册在 `router/index.ts` 中，并在 `Layout.vue` 的侧边栏自动生成导航项。

## 需求理解

### 功能边界
**包含功能：**
- [x] 新增“打标结果” (TaggedData) 页面组件。
- [x] 顶部提供强大的筛选栏（根据标签、批次、打标状态等进行检索）。
- [x] 中央使用 `el-table` 展示已打标数据，重点高亮展示“标签”列（使用彩色 `el-tag`）。
- [x] 提供导出结果的功能按钮。
- [x] 将该页面集成到现有的路由与左侧菜单中。
- [x] 更新 `UI_DESIGN.md`，将此页面补充进核心页面规划中。

**明确不包含（Out of Scope）：**
- [x] 后端数据库的实际查询逻辑（前端暂时使用 mock 数据或空壳组件，保持与现有重构进度一致）。
- [x] 复杂的全文搜索引擎集成。

## 疑问澄清
- **Q**: 新页面的菜单图标使用什么？
  **A**: 建议使用 Element Plus 的 `DataBoard` 或 `Collection` 图标。
