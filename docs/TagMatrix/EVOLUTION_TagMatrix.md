# 需求演进与架构决策记录 - TagMatrix

本文档用于记录 TagMatrix 系统在开发各阶段经历的需求变更、演进以及我们应对这些变化的架构和设计决策。

---

## 阶段：V1.0 初始核心架构对齐
**日期**：2026-04-23
**事件**：初次需求梳理与技术栈确立。

### 需求概述
- 开发一个通用的数据打标系统。
- 具有 UI，支持可视化自定义标签和打标规则。
- 数据源支持 CSV/Excel/数据库等。
- 引入 AI 模型（OpenAI 协议）辅助打标。

### 核心决策与解决方案
1. **产品形态选择**：决定放弃传统的 Web B/S 架构（需要额外部署），转而使用 **Wails (Go + Vue3)** 构建跨平台的本地单体桌面应用。
2. **本地数据中心**：为了支撑数百万行数据的处理和频繁打标，放弃单纯的“内存映射”，转而采用 **SQLite + GORM**，将导入的文件集中入库，在本地构建一个轻量级数据中心。
3. **复用 matcher 引擎**：决定复用原有 `NeoScan` 项目中的高性能无状态 `matcher` 引擎，以支持复杂的嵌套布尔逻辑。
4. **AI 隔离**：AI 不与基础的规则引擎强耦合，而是作为一个**独立的任务流**存在（即“规则首发过滤，AI长尾兜底”），保证可控性和效率。

## 阶段：V1.1 核心需求深度扩展
**日期**：2026-04-23
**事件**：针对打标模式、AI 能力、追踪审计、版本控制提出了更深度的需求。

### 需求变更点
1. **多种打标模式**：新增支持单标签、多标签、混合模式（主副标签）。
2. **多字段匹配**：规则不再局限单一字段，支持多个字段内容的联合匹配。
3. **全方位 AI 助手**：AI 能力不仅仅局限于“打标”，还要求可以像 ChatGPT 一样与用户对话，辅助查看/筛选数据、分析数据、生成 SQL 语句，并能作为软件使用说明书。
4. **配置可迁移性**：标签体系、打标规则需要支持 CSV 和 JSON 的导入导出。
5. **审计追踪与回退（版本控制）**：
   - 增加详尽的打标日志（包含打标理由、触发的规则、计算方式）。
   - 增加任务批次记录。
   - **支持任务回退（Rollback）**，允许用户在不满意时撤销某次打标结果。
6. **防误操作机制**：支持规则算子设置时的“试运行（Dry Run）”预览功能。

### 架构应对与解决方案
1. **数据模型扩展**：
   - 实体-标签关联表 (`SysEntityTag`) 需要增加字段来区分打标模式（如 `is_primary` 字段用于区分混合模式中的主副标签）。
   - 增加 `TagTaskBatch`（任务批次表）和 `TagTaskLog`（详尽日志表），记录每次打标任务的快照，为 Rollback（回退）提供数据支持。
2. **规则引擎扩展（Dry Run）**：
   - 在前后端交互中，新增 `/api/rules/dry_run` 接口，允许前端传递算子和少量采样数据（如 LIMIT 10），后端运行 `matcher` 引擎并立刻返回命中结果给前端，而不实际落盘。
3. **AI 聊天室模式**：
   - 在 Vue3 前端新增一个常驻的“AI 智能助手”侧边栏或独立页面。
   - 在 Prompt 设计中，向大模型注入当前 SQLite 的 Table Schema 和表结构上下文，使其具备“生成精确 SQL”的能力。
4. **底层架构预留**：虽然当前以 SQLite 为主，但通过 GORM 接口预留多数据源抽象，为未来支持 MySQL/PostgreSQL 做好准备。

## 阶段：V2.0 核心功能完善与数据看板重构
**日期**：2026-04-25
**事件**：打标任务管理闭环、重构打标数据展示看板，并修复统计与显示相关的核心缺陷。

### 需求变更与新增功能
1. **真实数据源过滤**：放弃之前按“导入批次”进行打标范围选择的逻辑，改为直接按用户原始数据中的“数据来源”字段进行过滤，更加符合业务直觉。
2. **打标数据看板重构**：
   - 彻底废弃仅展示原始表结构的做法，改为动态解析并展示所有导入数据的字段。
   - 新增展示系统处理字段（打标模式、命中标签、命中主标签、任务批次、状态等），支持全字段过滤检索。
   - 增加数据导出功能（CSV 格式），且自动排除系统内置的 ID 和打标时间字段。
   - 支持动态列的显示/隐藏（带滚动条的小齿轮配置），并统一了与【数据源管理】一致的 JSON 详情预览弹窗。
3. **任务历史管理闭环**：
   - 实现【查看日志】真实功能，提供弹窗展示详尽的每次打标审计记录。
   - 实现任务日志的 CSV 导出功能，解决 Excel 打开乱码问题（加入 UTF-8 BOM）。
   - 增加多选批量“硬删除”功能，清理冗余或测试任务记录。
4. **UI 与体验调优**：
   - 优化了【打标任务看板】按钮的排列布局与样式颜色，增加了功能说明 Tooltip。
   - 在左侧导航栏（`Layout.vue`）底部增加了软件版本与版权信息（可折叠隐藏）。
   - 在【数据源管理】页添加了“导入重复数据”的逻辑说明小问号。

### 架构应对与解决方案
1. **JSON 字段动态提取（SQLite）**：
   - 利用 SQLite 的 `json_extract(data, '$."字段名"')` 函数直接在数据库层面实现基于动态 JSON 列（如 `数据来源`）的 `GROUP BY` 聚合查询与 `WHERE` 过滤，避免了将全量数据加载到内存中解析。
2. **精准的 GORM 去重统计与软删除关联处理**：
   - 修复了概览控制台中“打标覆盖率超过 100%”的缺陷。原因是原有的 `.Select("count(distinct(record_id))").Count()` 会被 GORM 内部逻辑覆盖而变成统计总条数（包含同一记录的多个标签）。
   - **解决方案**：改为使用 `.Distinct("sys_entity_tags.record_id").Count()`，并通过 `.Joins` 关联 `raw_data_records` 表，加上 `.Where("raw_data_records.deleted_at IS NULL")` 条件，确保已软删除的数据不再被计入覆盖率中。
3. **彻底的级联硬删除 (`Unscoped`)**：
   - 对于任务历史的删除功能，使用了 GORM 的 `Unscoped()` 开启事务进行硬删除，确保删除任务批次时，其产生的 `SysEntityTag` 和 `TagTaskLog` 被一并从数据库物理移除，防止脏数据干扰后续的统计分析。
4. **动态列与系统列的融合渲染**：
   - 前端采用 Vue3 响应式变量（`visibleColumns` / `hiddenColumns`），结合数据解析预处理阶段提取的 Keys 集合，实现了动态列控制。
   - 在后端数据导出时，同样先遍历一次全量分页数据收集所有的 Key（剔除敏感字段），再拼接系统生成的关联字段，构建完整的表头与数据行。
