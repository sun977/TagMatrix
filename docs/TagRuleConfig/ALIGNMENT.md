# 需求对齐文档 - TagRuleConfig 多级标签与规则重构

## 原始需求
1. 【标签与规则配置】页面中的标签和规则都是假的数据，需要换成真实数据
2. 【标签与规则配置】中的标签支持多层级创建和删除，目前展示数据都不满足，同时还需要支持标签json结构数据的导出和导入
3. 匹配规则是支持多级结构的，可以通过前端的配置最终生成一段json数据，样例参考 `匹配规则.json`。前端需支持配置该多级结构，并提供一个按钮弹出直接查看生成的 json 匹配规则。

## 项目上下文

### 现有架构理解
- 前端：Vue 3 + Element Plus (Composition API)
- 后端：Go + Wails + GORM (SQLite)
- 现存组件：`TagRuleConfig.vue` 包含一个左侧的标签树（目前只有静态渲染或单层级添加），右侧包含表单。
- 数据库模型：`SysTag` 支持 `ParentID`，天然支持多级树结构。`SysMatchRule` 支持 `RuleJSON` 存储规则序列化结果。

## 需求理解

### 功能边界
**包含功能：**
- [x] 后端 API 补齐（多级标签树形数据返回、删除多级标签、导出/导入标签 JSON 文件）
- [x] 前端左侧标签树改造（支持在指定节点下新增子标签、删除标签及子标签、提供导入/导出按钮）
- [x] 前端右侧匹配规则构建器改造（支持多级逻辑组 `AND/OR` 的嵌套，支持各种操作符 `contains, equals, regex` 等）
- [x] 右侧增加“预览JSON”按钮，一键展示 `NeoScan` 格式的 JSON

## 疑问澄清

### 默认决策
- **标签导入策略**：当导入的标签名称冲突时，采用“存在则跳过（保留原节点及其子节点关系），不存在则新增并重建父子关系”的策略。
- **规则格式兼容**：确保最终生成的 JSON 结构完全符合提供的 `NeoScan/匹配规则.json` 样例结构，以便后续 `aiengine` 或 `taglogic` 解析使用。

## 验收标准
- [ ] 页面无 mock 数据，标签树完整映射数据库 `sys_tags` 表。
- [ ] 可在任意标签节点上右键（或悬浮按钮）添加子节点，并可删除。
- [ ] 点击导出，将标签树导出为 JSON 文件；点击导入，能选择 JSON 文件并成功合并入库。
- [ ] 右侧规则配置器支持“添加条件”与“添加条件组”，能自由构建 `AND/OR` 树。
- [ ] 点击“预览 JSON”，弹窗展示严格对应 `NeoScan` 格式的字符串。
